## 🚀 部署到 Render 時的資料處理

### ⚠️ 重要限制

**Render 免費方案不支持持久化文件系統**：
- 每次部署或重啟，文件系統會被重置
- 即使推送了客戶資料，在 Render 上也會丟失
- 資料只會保存在內存中，重啟後消失

### 解決方案

#### 選項 1：使用數據庫（推薦）

將客戶資料遷移到數據庫：
- **MongoDB Atlas**（免費）
- **PostgreSQL**（Render 提供）
- **Supabase**（免費）

#### 選項 2：每次部署後手動導入

1. 部署完成後，通過前端界面重新導入 CSV
2. 或使用 API 批量導入

#### 選項 3：使用環境變數（僅限少量資料）

對於少量客戶，可以使用環境變數（不推薦大量資料）
